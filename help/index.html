Viewing Logs

The first step in any diagnosis is checking the container logs. This often reveals the exact error message.

$ docker logs -f moltbot-app --tail 100

Rate Limit Exceeded (429)

If you see 429 Too Many Requests from OpenAI or Anthropic, you have hit your API quota. To resolve this:

  • Check your billing status on the provider's dashboard.
  • Implement exponential backoff in your requests (Moltbot handles this automatically by default).
  • Switch to a fallback model in your ROUTER_CONFIG.

Connection Refused

This usually means the Docker container isn't running or the port is blocked. Verify the container status with docker ps and ensure port 3000 is open in your firewall.

Updating Moltbot

To upgrade to the latest version of Moltbot without losing data, follow this standard procedure. Your database volume is persistent, so it will remain safe.

# Pull the latest images
docker-compose pull

# Restart containers (minimal downtime)
docker-compose up -d --remove-orphans

# Prune old images to save space
docker image prune -f

Database Management

Moltbot uses PostgreSQL. Regular backups are recommended before major updates.

# Create a backup
docker exec -t moltbot-db pg_dumpall -c -U moltbot > dump_$(date +%Y-%m-%d).sql

# Restore from backup
cat dump_2024-01-01.sql | docker exec -i moltbot-db psql -U moltbot

Frequently Asked Questions

How do I reset the admin password?
You can reset the password via the CLI tool inside the container: docker exec -it moltbot-app npm run reset-password <email>. Alternatively, you can modify the user record directly in the PostgreSQL database if you have access.
Why is my bot not responding on WhatsApp?
Check your webhook status in the Meta Developer Dashboard. Ensure your VPS firewall allows incoming traffic on the callback port (usually 3000 or 443 if proxied). Also verify that your WHATSAPP_VERIFY_TOKEN matches exactly.
