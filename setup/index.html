Moltbot Setup & Installation

Quick Summary

Setting up Moltbot requires a reliable environment to ensure 24/7 availability for your digital employees. This guide walks you through the complete installation process, starting from selecting the right hardware (Mac Mini or Linux VPS) to configuring the necessary runtime environments. We recommend a Docker-based deployment for maximum stability and security. By following these steps, you will have a fully functional sovereign AI agent ready to execute tasks on your behalf.

01 // Prerequisites

Before diving into the installation, ensure your infrastructure meets the following requirements to run Moltbot effectively. As a "sovereign" agent, Moltbot thrives on dedicated resources.

  • Hardware:
    • ➜ Option A (Recommended): Mac Mini (M2/M4 Pro) - Ideal for local model inference and ecosystem integration.
    • ➜ Option B (Cloud): Linux VPS (Ubuntu 22.04 LTS) with at least 4GB RAM and 2 vCPUs (e.g., DigitalOcean, Hetzner, AWS).
  • Software Environment:
    • ➜ Docker & Docker Compose: Essential for containerized deployment.
    • ➜ Node.js: Version 20.x or higher (LTS).
    • ➜ Git: For version control and updates.
  • API Keys: Valid API keys for your preferred LLM provider (Anthropic API Key for Claude 3.5/4.5 or OpenAI API Key for GPT-4o).

02 // Step 1: VPS Provisioning & SSH Access

If you are opting for a cloud deployment, choosing the right provider is crucial. We recommend DigitalOcean or Hetzner for their reliability and cost-effectiveness.

  1. Create a Droplet/Instance: Select an Ubuntu 22.04 LTS image. Choose a plan with at least 4GB RAM to handle memory-intensive operations like vector embeddings.
  2. Access via SSH: Open your terminal and connect to your server:
    ssh root@your-server-ip
  3. Secure Your Server (Crucial): Immediately create a non-root user, disable password authentication, and configure UFW (Uncomplicated Firewall) to allow only SSH (22), HTTP (80), and HTTPS (443).

03 // Step 2: Environment Configuration

Once logged in, update your system repositories and install the necessary dependencies. This ensures your OS is patched and ready for the Moltbot runtime.

# Update package list and upgrade system
sudo apt update && sudo apt upgrade -y

# Install Docker and Docker Compose
sudo apt install docker.io docker-compose -y

# Verify Docker is running
sudo systemctl status docker

04 // Step 3: Installation via Docker

The most reliable way to run Moltbot is using the official Docker image. This method isolates the application and manages dependencies automatically.

Clone the repository and build the container:

git clone https://github.com/moltbot/moltbot-core.git
cd moltbot-core
docker-compose build --no-cache

05 // Step 4: Configuring .env Variables

Configuration is managed via a .env file. Copy the example file to start:

cp .env.example .env
nano .env

Key variables you must configure:

  • MOLTBOT_API_KEY: Your secure gateway token. Treat this like a password.
  • ANTHROPIC_API_KEY: Required if using Claude models.
  • DATABASE_URL: PostgreSQL connection string (defaults to internal Docker container).

06 // Step 5: Launch & Verification

Start your Moltbot instance in detached mode:

docker-compose up -d

Verify the service is healthy by checking the logs or pinging the health endpoint. You should see "System Status: ONLINE" in the logs.

/// FAQ

01 // What is the minimum RAM requirement?
We recommend at least 4GB of RAM for a smooth experience, especially if you plan to use local embeddings or multiple heavy skills. 2GB may suffice for very basic text-only operations.
02 // Can I use Windows Server?
While possible via WSL2, we strongly recommend a native Linux environment (Ubuntu) or macOS for better compatibility with Docker and the Moltbot ecosystem tools.
