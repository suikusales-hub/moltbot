Connecting Channels

Quick Summary

One of Moltbot's most powerful features is its ability to live where your users are. Instead of forcing users to visit a specific website, Moltbot integrates directly into popular messaging platforms. This section covers the configuration for the three primary supported channels: WhatsApp, Discord, and Telegram. Whether you are building a community moderator for Discord or a customer support agent on WhatsApp, the unified channel architecture ensures your bot behaves consistently everywhere.

01 // WhatsApp Integration

Moltbot supports WhatsApp via the Moltbot WhatsApp Bridge, a local service that emulates a WhatsApp Web client. This method allows you to use your existing phone number without needing a Meta Business API account.

# In your docker-compose.yml
whatsapp-bridge:
  image: moltbot/whatsapp-bridge:latest
  environment:
    - MOLTBOT_CORE_URL=http://core:3000

Setup: Once the container is running, check the logs. It will generate a QR code. Scan this code with your WhatsApp mobile app (Linked Devices) to authenticate. Your bot will now listen to incoming messages and reply automatically.

02 // Discord Bot Setup

To connect Moltbot to Discord, you need to create an application in the Discord Developer Portal.

  1. Go to the Discord Developer Portal and create a "New Application".
  2. Navigate to the Bot tab and click "Add Bot".
  3. Important: Enable "Message Content Intent" under Privileged Gateway Intents. Without this, Moltbot cannot read user messages.
  4. Copy the Bot Token and add it to your .env file:
DISCORD_BOT_TOKEN=MTk4...

03 // Telegram Bot API

Telegram is the easiest channel to configure thanks to BotFather.

  • Open Telegram and search for @BotFather.
  • Send the command /newbot and follow the instructions to name your bot.
  • BotFather will give you an API Token (e.g., 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11).
  • Add this token to your Moltbot configuration. Moltbot will automatically set up the webhook to receive updates.

04 // Unified Messaging Protocol

Regardless of the channel, Moltbot normalizes all incoming events into a standard MessageReceived event. This means your skills and logic don't need to know if a message came from WhatsApp or Discord.

internal_event.json
{
  "type": "message",
  "content": "Hello Moltbot!",
  "sender": {
    "id": "user_123",
    "platform": "discord" // or "whatsapp", "telegram"
  },
  "timestamp": 1706523400
}

05 // Managing Multi-Channel Conversations

Moltbot uses Redis to maintain session state across channels. If you want to link a Discord user to a Telegram user (so they share the same memory), you can use the Identity Mapping feature in the admin dashboard. This allows a user to start a task on their desktop (Discord) and finish it on their phone (WhatsApp) without losing context.

/// FAQ

01 // Do I need a business account for WhatsApp?
It depends on the integration method. The Official WhatsApp Business API requires a Business account and verification. However, our "WhatsApp Bridge" method works with any standard personal WhatsApp account by emulating a web session.
02 // Can the bot be in multiple Discord servers?
Yes, a single Moltbot instance can serve multiple Discord servers simultaneously. You can configure whether the bot shares memory across servers or keeps each server's context isolated via the ISOLATE_GUILDS environment variable.
